# Lab 12: Network Firewall Configuration & Traffic Analysis 🕵️‍♂️

## 📋 Описание
Лабораторная работа по настройке межсетевого экрана `iptables`, проверке сетевой связности, DNS-разрешения, сканированию подсети через `nmap` и захвату трафика через `tcpdump`. Практика охватывает применение правил фильтрации, валидацию работы сервисов после изменения политик, анализ пакетов и трассировку маршрутов.

## 🎯 Цели
1. Настроить базовые параметры сети в тестовой среде
2. Проверить работоспособность веб-доступа и DNS до применения правил
3. Изучить текущий набор правил через `iptables-save`
4. Применить кастомные правила фильтрации/NAT
5. Проверить доступность доменов (vk.com, mail.ru, ya.ru) через `nslookup`, `ping` и браузер
6. Выполнить сканирование подсети `nmap 192.168.0.0/24`
7. Захватить и проанализировать трафик через `tcpdump`
8. Выполнить трассировку маршрутов (`tracert`/`traceroute`) до и после изменений
9. Сравнить поведение сети при разных политиках iptables

## 🛠 Выполненные задания

### Этап 1: Базовая настройка и проверка связности
- Настроены сетевые параметры в Windows/Linux-среде
- Подтверждена работоспособность: сайты открываются в браузере
- Выполнены базовые тесты: `ping`, `nslookup` до целевых доменов

### Этап 2: Настройка межсетевого экрана (iptables)
- Выведен текущий набор правил: `iptables-save`
- Применены правила фильтрации (INPUT/OUTPUT/FORWARD, политики по умолчанию)
- Проверено применение правил без потери управления (сохранён доступ по SSH/RDP)
- Проведена повторная валидация:
  - `nslookup` и `ping` для vk.com, mail.ru, ya.ru
  - Открытие сайтов в браузере
  - Фиксация изменений в поведении сети

### Этап 3: Сканирование и анализ трафика
- Запущено сканирование подсети: `nmap 192.168.0.0/24`
- Проанализированы результаты: открытые порты, версии сервисов, ОС-фингерпринты
- Инициирован захват пакетов: `tcpdump -i <interface> -w lab12.pcap`
- Сгенерирован тестовый трафик (HTTP-запросы, ping, DNS-запросы)
- Изучен вывод `tcpdump`: структура пакетов, протоколы, флаги, соответствие правилам iptables

### Этап 4: Трассировка и финальная валидация
- Выполнена трассировка маршрутов: `tracert`/`traceroute` до целевых хостов
- Повторена трассировка после применения новых правил
- Сравнены количество хопов, задержки, точки фильтрации
- Подтверждена стабильность маршрутизации и корректность работы firewall

## 📊 Ключевые выводы

| Инструмент | Назначение | Ключевой инсайт |
|------------|------------|-----------------|
| **iptables** | Фильтрация, NAT, логирование | Порядок правил критичен; политики по умолчанию (ACCEPT/DROP) формируют базовый периметр |
| **tcpdump** | Захват и анализ пакетов | Даёт «сырую» картину трафика; необходим для верификации, что правила iptables работают как задумано |
| **nmap** | Аудит сети и безопасности | Выявляет открытые порты, запущенные сервисы, потенциальные векторы атаки |
| **nslookup / ping** | Диагностика DNS и L3-связности | Базовые инструменты для изоляции проблем: имя → IP → доступность |
| **tracert / traceroute** | Диагностика маршрутизации | Показывает путь пакетов, задержки на хопов, точки блокировки или фильтрации |

### Почему это важно для сисадмина:
✅ **Безопасность периметра**: грамотные правила iptables сокращают attack surface  
✅ **Видимость трафика**: tcpdump позволяет отладить сложные сценарии (NAT, пробросы, Drop-правила)  
✅ **Проактивный аудит**: nmap помогает поддерживать инвентарь и выявлять «забытые» сервисы  
✅ **Навыки диагностики**: ping/nslookup/tracert — фундамент L3-L4 troubleshooting в любой инфраструктуре  

## 🔧 Использованные инструменты
- `iptables` / `iptables-save` — управление правилами фильтрации и NAT
- `tcpdump` — захват и анализ сетевых пакетов (BPF-фильтры, экспорт в PCAP)
- `nmap` — сканирование портов, версий сервисов, ОС-детекция
- `nslookup` / `dig` — прямые DNS-запросы
- `ping` / `tracert` / `traceroute` — проверка связности и маршрутов
- Web Browser — валидация на прикладном уровне (L7)
- Astra Linux / Windows — тестовые среды

## 📚 Теоретическая база

### Архитектура iptables
Работает с цепочками (INPUT, OUTPUT, FORWARD) и таблицами (filter, nat, mangle, raw). Правила обрабатываются сверху вниз, срабатывает первое совпадение. Политики по умолчанию (`-P`) определяют судьбу пакетов, не попавших ни под одно правило. Критичен для реализации defense-in-depth.

### tcpdump и анализ пакетов
Захватывает сырые кадры на уровне сетевого интерфейса. Использует синтаксис Berkeley Packet Filter (BPF) для выборки. В выводе отображаются временные метки, IP-адреса, порты, протоколы, TCP-флаги и payload (если не зашифрован). Незаменим для отладки firewall-правил и диагностики сетевых аномалий.

### Техники сканирования nmap
Использует TCP SYN, TCP Connect, UDP-зонды и OS-фингерпринтинг. Флаги `-sV` определяют версии сервисов, `-O` — ОС, `-sC` — запускает стандартные скрипты. В enterprise-среде применяется для compliance-проверок и картирования поверхности атаки.

### DNS и трассировка маршрутов
`nslookup` обращается напрямую к DNS-серверу, минуя локальный кэш. `tracert`/`traceroute` использует истечение TTL (ICMP/UDP/TCP) для построения карты хопов. В связке они изолируют природу сбоев: DNS-ошибка, маршрутизация или блокировка firewall.
