# Lab 7: Astra Linux Networking, SSH Hardening & Firewall/NAT Configuration 🛡️

## 📋 Описание
Лабораторная работа по настройке безопасного удалённого доступа, маршрутизации, межсетевого экранирования и автоматизированной раздачи сетевых параметров в Astra Linux.

## 🎯 Цели
1. Настроить SSH-сервер для безопасного удалённого управления
2. Реализовать аутентификацию по SSH-ключам через PuTTY
3. Включить IP-форвардинг ядра для маршрутизации трафика
4. Настроить NAT/Masquerading через UFW и iptables
5. Развернуть DHCP-сервер для автоматической раздачи IP/DNS клиентам
6. Проверить связность и разрешение имён с Windows-клиента

## 🛠 Выполненные задания

### Задание 1: Настройка SSH и ключевой аутентификации
**Конфигурация сервера:**
- Отредактирован `/etc/ssh/sshd_config` (отключён вход по паролю, включён `PubkeyAuthentication`)
- Перезапущен сервис: `sudo systemctl restart ssh`
- Сгенерирована пара ключей (RSA/Ed25519)
- Публичный ключ добавлен в `~/.ssh/authorized_keys`

**Клиентская часть (PuTTY):**
- Импортирован приватный ключ в PuTTYgen → `.ppk`
- Настроена сессия: IP, порт 22, SSH-2, Auto-login username
- Подключение без пароля успешно проверено
