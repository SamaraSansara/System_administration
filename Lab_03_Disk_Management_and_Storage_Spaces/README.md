# Lab 3: Disk Management and Storage Spaces 💾

## 📋 Описание
Лабораторная работа по управлению дисками и созданию дисковых пространств (Storage Spaces) в Windows Server. Практическое освоение различных типов томов и уровней отказоустойчивости.

## 🎯 Цели
1. Освоить создание различных типов томов через управление дисками
2. Изучить работу с дисковыми пространствами (Storage Spaces)
3. Понять различия между аппаратными и программными RAID-массивами
4. Научиться проверять тома на отказоустойчивость

## 🛠 Выполненные задания

### Задание 1: Работа с управлением дисками

#### 1.1 Чередующийся том (Striped Volume / RAID 0)
**Инструменты:** Disk Management, Disk 1 + Disk 2

**Выполненные операции:**
- Добавление в него диска (физический сетевой диск)
- Проверка на отказ: запуск «Форматировать раз»
- Запуск «Поганить раз» (проверка устойчивости)

**Характеристики:**
- Тип: Striping (RAID 0)
- Производительность: Высокая (параллельная запись/чтение)
- Отказоустойчивость: Отсутствует
- Минимум дисков: 2

#### 1.2 Зеркальный том (Mirrored Volume / RAID 1)
**Инструменты:** Disk Management, Disk 1 + Disk 2

**Выполненные операции:**
- Добавление в него диска (физический сетевой диск)
- Запуск «Форматировать раз»
- Удаление отказа зеркального
- Проверка доступности карты
- Добавление свободного диска в качестве резервного
- После ресинхронизации проверка отказа второго диска («Форматировать два»)
- Проверка доступности карты

**Характеристики:**
- Тип: Mirroring (RAID 1)
- Производительность: Средняя (запись на оба диска)
- Отказоустойчивость: Выдерживает отказ 1 диска
- Минимум дисков: 2
- Полезное пространство: 50% от общего

#### 1.3 Составной том (Spanned Volume)
**Инструменты:** Disk Management, Disk 1 + Disk 2

**Выполненные операции:**
- Добавление в него диска (физический сетевой диск)
- Проверка на отказ: запуск «Форматировать раз»
- Запуск «Поганить раз»

**Характеристики:**
- Тип: Spanning (JBOD)
- Производительность: Базовая
- Отказоустойчивость: Отсутствует
- Использование пространства: Последовательное заполнение

### Задание 2: Работа с дисковыми пространствами (Storage Spaces)

#### 2.1 Простое пространство (Simple Space)
**Инструменты:** Storage Spaces через «Дисковые пространства»

**Выполненные операции:**
- Добавление в него диска (физический сетевой диск)
- Проверка на отказ одного диска
- Проверка доступности карты (скриншот проводника)
- Проверка на отказ двух дисков
- Проверка доступности карты

**Характеристики:**
- Тип: Simple (без отказоустойчивости)
- Аналог: RAID 0
- Производительность: Максимальная
- Отказоустойчивость: Нет

#### 2.2 Двустороннее зеркало (Two-way Mirror)
**Инструменты:** Storage Spaces

**Выполненные операции:**
- Добавление в него диска (физический сетевой диск)
- Проверка на отказ одного диска
- Проверка доступности карты (скриншот проводника)
- Проверка на отказ двух дисков
- Проверка доступности карты

**Характеристики:**
- Тип: Two-way Mirror
- Аналог: RAID 1
- Производительность: Хорошая для чтения
- Отказоустойчивость: Выдерживает отказ 1 диска
- Минимум дисков: 2
- Полезное пространство: 50%

#### 2.3 Пространство с четностью (Parity Space)
**Инструменты:** Storage Spaces

**Выполненные операции:**
- Добавление в него диска (физический сетевой диск)
- Проверка на отказ одного диска
- Проверка доступности карты (скриншот проводника)
- Проверка на отказ двух дисков
- Проверка доступности карты

**Характеристики:**
- Тип: Parity Space
- Аналог: RAID 5
- Производительность: Хорошая для чтения, средняя для записи
- Отказоустойчивость: Выдерживает отказ 1 диска
- Минимум дисков: 3
- Полезное пространство: (N-1)/N от общего

## 🔑 Ключевые выводы

### Управление дисками vs Дисковые пространства

**Управление дисками (Disk Management):**
- Простота использования (GUI)
- Быстрое создание томов
- Ограниченная гибкость
- Требует простои для изменений

**Дисковые пространства (Storage Spaces):**
- Гибкость в управлении
- Возможность горячего добавления дисков
- Автоматическая ресинхронизация
- Поддержка тонкого выделения (thin provisioning)
- Требует больше ресурсов системы

### Отказоустойчивость

**Зеркальные тома** обеспечивают наилучшую защиту данных при отказе одного диска  
**Пространства с четностью** оптимальны для архивных данных (хорошее соотношение容量/надежность)  
**Простые и чередующиеся тома** не обеспечивают отказоустойчивость — только для временных/некритичных данных

### Практическое применение

| Сценарий | Рекомендуемый тип |
|----------|-------------------|
| Временные файлы, кэш | Striped (RAID 0) |
| Системный диск, базы данных | Mirrored (RAID 1) |
| Файловый сервер, архивы | Parity Space (RAID 5) |
| Тестовые среды | Simple |

## 🔧 Использованные инструменты

- **Disk Management (diskmgmt.msc)** — управление дисками Windows
- **Storage Spaces** — программно-определяемые хранилища Windows
- **DiskPart** (опционально) — консольная утилита
- **File Explorer** — проверка доступности данных
- **Performance Monitor** — мониторинг производительности

## 📚 Теоретическая база

### RAID-уровни в Windows:

- **RAID 0 (Striping)** — распределение данных без избыточности
- **RAID 1 (Mirroring)** — зеркалирование данных
- **RAID 5 (Parity)** — распределение с четностью
- **JBOD (Spanning)** — простое объединение дисков

### Дисковые пространства (Storage Spaces):

Программная реализация RAID в Windows Server и Windows 10/11, позволяющая:
- Объединять физические диски в пулы
- Создавать виртуальные диски с различными уровнями устойчивости
- Динамически изменять размер и конфигурацию
- Автоматически восстанавливать данные после замены диска
