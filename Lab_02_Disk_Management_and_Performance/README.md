## 📊 Сравнительный анализ

### Типы томов и их характеристики

| Тип тома | Количество дисков | Преимущества | Недостатки |
|----------|-------------------|--------------|------------|
| **Простой (Simple)** | 1 | Простота, надежность | Нет отказоустойчивости, базовая производительность |
| **Перекрестный (Striped/RAID 0)** | 2-3 | Высокая скорость чтения/записи | Нет отказоустойчивости, потеря всех данных при отказе одного диска |
| **Составной (Spanned)** | 2+ | Объединение пространства | Нет прироста производительности, риск потери данных |

### Ключевые выводы

**CrystalDiskMark** — стандарт индустрии для тестирования производительности накопителей  
**Перекрестные тома (RAID 0)** показывают наилучшую производительность, особенно на 3 дисках  
**Простые тома** — наиболее надежны для критичных данных  
**Составные тома** удобны для объединения дисков разного размера  
**Управление дисками Windows** предоставляет удобный GUI для всех операций

## 🔧 Использованные инструменты

- **DiskPart** (опционально) — консольная утилита управления дисками

## 📚 Теоретическая база

### Типы томов в Windows:

1. **Простой том (Simple Volume)** — использует пространство одного диска
2. **Составной том (Spanned Volume)** — объединяет пространство нескольких дисков
3. **Перекрестный том (Striped Volume, RAID 0)** — распределяет данные между дисками для повышения производительности
4. **Зеркальный том (Mirrored Volume, RAID 1)** — дублирование данных (требует Windows Server)
5. **RAID-5** — распределение с четностью (требует Windows Server)
