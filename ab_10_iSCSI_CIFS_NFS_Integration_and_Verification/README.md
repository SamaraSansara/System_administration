# Lab 10: iSCSI, CIFS & NFS Integration + Verification Workflow 🗄️

## 📋 Описание
Итоговая лабораторная работа по комплексной настройке и проверке работы сетевых хранилищ в Astra Linux. Практика включает создание и маппинг LUN через iSCSI, организацию файловых шаров CIFS/SMB и NFS, создание пользователей, монтирование ресурсов и верификацию доступности через системные утилиты.

## 🎯 Цели
1. Развернуть инфраструктуру сетевых хранилищ на 5 виртуальных машинах
2. Настроить iSCSI Target/Initiator и выполнить маппинг LUN
3. Организовать кросс-платформенный доступ через CIFS (Windows) и NFS (Linux)
4. Создать учётные записи и настроить права доступа к шарам
5. Выполнить монтирование файловых систем и проверить занимаемое пространство (`df`, `lsblk`)
6. Провести полную верификацию доступности всех компонентов

## 🛠 Выполненные задания

### Задание 1: Создание LUN (Logical Unit Number)
- Инициализирован дисковый ресурс в системе хранения
- Присвоен уникальный IQN/идентификатор LUN
- Настроен режим выделения (thin/thick provisioning)

### Задание 2: Создание инициатора iSCSI
- Установлен и настроен пакет `open-iscsi`
- Сконфигурирован `/etc/iscsi/iscsid.conf` (InitiatorName, параметры аутентификации)
- Запущен сервис: `sudo systemctl enable --now iscsid`

### Задание 3: Создание Host (клиентского узла)
- Зарегистрирован Host в target-системе по IQN
- Настроены ACL (Access Control List) для доступа к LUN

### Задание 4: Маппинг LUN на Host
- Выполнено связывание LUN ↔ Host
- Проверена видимость тома на стороне инициатора
