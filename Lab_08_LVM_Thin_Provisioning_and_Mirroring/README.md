# Lab 8: LVM Thin Provisioning & Mirroring in Astra Linux 💾

## 📋 Описание
Лабораторная работа по продвинутому управлению дисковой подсистемой в Astra Linux с использованием LVM2. Практика охватывает создание зеркальных томов, настройку thin provisioning (тонкого выделения пространства), монтирование файловых систем и мониторинг состояния дисков.

## 🎯 Цели
1. Освоить базовые команды LVM для управления Physical Volumes (PV), Volume Groups (VG) и Logical Volumes (LV)
2. Создать и протестировать зеркальный том (LVM Mirror) для повышения отказоустойчивости
3. Настроить thin provisioning для экономии дискового пространства
4. Автоматизировать монтирование через `/etc/fstab`
5. Научиться диагностировать состояние дисков и томов через `df`, `lvs`, `uptime`

##  Выполненные задания

### Задание 1: Подготовка среды
- Проверка/установка пакета `thin-provisioning-tools`
- Создание физического тома (PV) и группы томов (VG) под лабораторную работу

### Задание 2: Создание зеркального тома (Mirror)
- Создан зеркальный логический том через `lvcreate -m1`
- Просмотр параметров тома: уровень избыточности, распределение по PV, статус синхронизации
- Проверка отказоустойчивости: эмуляция сбоя одного из физических дисков

### Задание 3: Создание файловой системы и монтирование
- Форматирование тома в `ext4`/`xfs` через `mkfs`
- Создание точки монтирования и ручное монтирование (`mount`)
- Редактирование `/etc/fstab` для автоматического монтирования при загрузке
- Проверка корректности записи через `mount -a`

### Задание 4: Настройка Thin Provisioning
- Создан thin pool (пул тонкого выделения) внутри VG
- Создано 5 тонких логических томов (thin LV) поверх пула
- Примонтированы файловые системы тонких томов
- Проверка перераспределения пространства: тонкие тома занимают место только по мере записи данных

### Задание 5: Мониторинг и диагностика
- Выполнена команда `df -h` — проверка используемого и свободного пространства на смонтированных ФС
- Выполнена команда `lvs` — просмотр статуса всех логических томов (размер, тип, pool, данные/мета)
- Выполнена команда `uptime` — контроль нагрузки на систему после операций с дисками
- Сравнение вывода `lvs` до и после создания 5 тонких томов

## 📊 Ключевые выводы

| Технология | Назначение | Плюсы | Минусы/Риски |
|------------|------------|-------|--------------|
| **LVM Mirror** | Отказоустойчивость на уровне томов | Высокая надёжность, прозрачная замена дисков | Требует 2x дискового пространства, снижение скорости записи |
| **Thin Provisioning** | Экономия места, гибкое выделение | Экономия до 70-90% места, быстрое создание томов | Риск переполнения пула (требуется мониторинг `lvs`/`dmsetup`) |
| **/etc/fstab** | Авто-монтирование при загрузке | Стабильность, отсутствие ручных действий после ребута | Ошибка в синтаксисе → невозможность загрузки (требуется recovery mode) |

### Почему это важно для сисадмина:
**Гибкость**: LVM позволяет менять размер томов «на лету» без потери данных  
**Безопасность**: Зеркалирование защищает от аппаратных сбоев дисков  
**Экономия**: Thin provisioning оптимизирует использование дорогих SSD/NVMe в серверах  
**Автоматизация**: `/etc/fstab` + скрипты мониторинга = предсказуемая инфраструктура  

## 🔧 Использованные инструменты
- `lvm2`, `thin-provisioning-tools` — управление томами и тонким выделением
- `lvcreate`, `vgcreate`, `pvcreate` — создание структур LVM
- `mkfs.ext4` / `mkfs.xfs` — создание файловых систем
- `/etc/fstab`, `mount`, `umount` — монтирование и автозагрузка
- `df -h`, `lvs`, `vgs`, `pvs`, `uptime` — диагностика и мониторинг
- Astra Linux (Debian-based) — целевая ОС

## 📚 Теоретическая база

### Архитектура LVM:
1. **Physical Volume (PV)** — физический диск или раздел, инициализированный для LVM
2. **Volume Group (VG)** — пул пространства из одного или нескольких PV
3. **Logical Volume (LV)** — виртуальный раздел, создаваемый внутри VG (то, что форматируется и монтируется)

### Thin Provisioning:
Технология, при которой логическому тому выделяется пространство **виртуально**, а физически место резервируется только при записи данных. Позволяет создавать тома размером больше, чем доступно физически, но требует строгого контроля заполнения thin pool.

### Зеркалирование в LVM:
Реализуется через `lvcreate -m1`. Данные пишутся одновременно на два физических диска. При сбое одного из них LVM автоматически переводит том в degraded mode, сохраняя доступность данных.
